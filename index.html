Digital Piano (2024)

The piano was built from scratch with laser cut wood and springs. We used Hall Effect sensors and ADCs to measure how far each key was pressed down and detected how many were pressed at a time. This information was used to modulate the addition and amplitude of notes accordingly, before sending them out to play through a DAC or using a bluetooth speaker depending on the selected mode.

24 piano notes sampled at 11kHz were stored on an SD card, and was able to be switched out with another SD card to hear an organ instrument instead.

Tic-Tac-Toe-Playing Delta Robot (Against Human) (2023)

This is a tictactoe-playing robot that I made. All the moves are hard coded (so they don't always line up perfectly with the grid). That involved using 6 buttons to control the 3 motors forward/backward until the pen ended up in the desired location on the grid. Then I saved all those moves so that I could play them back whenever I needed to go back to that grid location. While this method avoided the complicated delta robot kinematics, it involved some error especially because the docked/starting position of the robot could not always be exactly the same.

Uses 3 stepper motors, raspberry pi, and webcam. Mechanism is inspired by delta robot and uses some 3d printed parts I designed, as well as other parts from robotics kits. Programmed using python and uses OpenCV for color detection of the opponent's moves. The game strategy itself uses minimax algorithm.

While I first attempted to create an object detection model to identify X's and O's, it was difficult to obtain enough control on the robot pen to draw these shapes. Color detection proved to be a good alternative.
